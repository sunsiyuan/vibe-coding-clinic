## 01｜今天连线的观众：芋头是谁？

「芋头」是一个**都市小说深度爱好者**，日常写**都市短篇**，偏**惊悚主线**，但会在暗线里埋"现实警醒"的主题。  
她的写作方式也高度依赖 AI：从大纲到正文再到修订，几乎全流程都在与模型协作。

---

## 02｜芋头的 3 个核心难点（也是很多 AI 写作者的共性痛点）

### 1）因果 / 常识缺失（读者一秒出戏）
- 例 1：十字路口剧情里，救护车多等了 2 个红绿灯 → 患者死亡（现实中救护车不会这么等）
- 例 2：主角因晋升通道受阻 → 直接赌博（动机、契机断层）

### 2）行文风格与世界观冲突（AI 容易"混写"）
- 民俗恐怖的词汇，硬塞到赛博科幻里
- 同一篇里风格漂移，读者无法建立稳定的沉浸感

### 3）可读性差（黑话 + 连接词重复）
- 常见黑话：抽离、屏障、非自然形态转换……
- 承上启下的词汇、句式容易反复，读起来像"模板拼接"

---

## 03｜芋头目前的解决方案（努力但痛苦）

她已经在用"工程化"方式自救：
- 单篇修订：约 **半小时/篇**
- 建黑词库、做 SOP
- SOP 流程：大纲 → 正文 → 黑话替换 → 人工修订 → 按平台规则（番茄/微信）排版发布

但问题是：**做得越认真，越像在"对抗 AI 的随机性"**，写作体验持续痛苦。

---

## 04｜我今天的"不同思路"到底不同在哪里？

我没有从"怎么把 AI 写得更像人"入手，而是从三个更底层的方向切：

### A. 先修"因果骨架"，再谈文笔
很多 AI 文的问题不在"词不好看"，而在"读者不信"。  
所以我优先让模型做的是：**找漏洞、补常识、补动机、补触发点**。

### B. 把写作拆成"可复用模块"，而不是一篇篇硬修
芋头现在是"对每篇文章做一次手工救火"。  
我做的是把火源拔掉：把关键设定沉淀成 **世界观与元素库**，让后续写作变成"装配"，而不是"反复返工"。

### C. 用多轮 prompt 形成一条稳定流水线（而非一次性生成）
不是寄希望于"一个神 prompt 生成神文"，而是像做产品一样：
- 先定义目标与结构
- 再压缩逻辑与字数预算
- 最后做可读性 QA（统计 + 举例 + 定点修）

## 05｜直播现场：我带芋头跑了一遍"AI 写小说的正确姿势"

> 方式：轮换使用 Cursor 的 ask / agent 模式；主要执行由 Sonnet 4.6 完成  
> 前提：我先向芋头要了她最近一篇文章的大纲（作为她风格与方法的"基准样本"）

### Prompt 1：让模型复述"选题 + 行文思路"
- 输入：《红白信号灯的偏执》的大纲（含立意、角色档案、剧情蓝图、执行卡点）
- 产出：快速抓住主题隐喻与结构链路（罪→遮掩→回归→异象→物化→符号化）

✅ 目的：先确认"芋头的写作骨架是清晰的"，问题主要在 AI 协作方式，而不是她没想法。

---

### Prompt 2-3：同框复刻能力 + 剧情逻辑"砍刀最小"
- Prompt 2：生成新选题《送风口的沉默》（立意/角色/蓝图齐全）
- Prompt 3：把剧情逻辑压到极简版（可审、可改、可补）

✅ 目的：把复杂创作压缩成"可校验的逻辑卡片"。

---

### Prompt 4：动机、因果链、现实常识的"体检报告"
针对极简剧情，模型给出最大缺口与补齐优先级：
- 回扣动机逻辑不通
- "缺氧猝死"现实条件太苛刻
- 六年时间段人物心理是空白
- 审计为何会翻到六年前数据
- 为何必须当晚亲自到场

✅ 这一步是今天最关键的转折点：  
**不是"写得不恐怖"，而是"读者不信"。先让读者信，恐怖才成立。**

---

### Prompt 5：把补齐内容沉淀为《世界观与元素库》（模块化）
输出为一个可复用的 markdown 文档，包括：
- 世界观底色（写字楼像"有内脏的身体"）
- 人物画像（三人完整动机与阶段变化）
- 犯罪结构（回扣链条 + 数据为何难删）
- 恐怖机制（递进四级：系统→身体→空间→存在）
- 叙事节奏建议
- 意象词典（可直接做写作锚点）

✅ 目的：从此以后写作不是"修一篇"，而是"复用一套系统"。

---

### Prompt 6-8：按平台字数限制做"结构取舍"并落地成文
考虑平台约束：
- 番茄：5000 字限制
- 微信：4000 字限制

Prompt 7：让模型给出"取舍决策"与字数预算  
Prompt 8：按取舍生成全文（最终约 **2424 字**，落在 4000 内的合理区间）

✅ 目的：把"好看"变成"可发布"。

---

### Prompt 9-10：可读性 QA + 句式转接精修
从四个维度逐项检查：
- 用词与世界观匹配度
- 黑话/通俗用语比例
- 分行分段与阅读节奏
- 句式转接重复与改写点

然后 Prompt 10 做定点修复，完成最终版本。

✅ 目的：让 AI 从"会写"走到"能发表"。

---

## 06｜今天这场急诊，给到芋头的"最终收益"

- **从"逐篇救火" → "搭系统"**：世界观与元素库是最值钱的资产
- **从"靠灵感" → "靠结构"**：因果链 + 常识体检，让读者先信再怕
- **从"AI 乱写" → "你指挥 AI 写"**：多轮 prompt 流水线，把随机性关进笼子里
- **从"写作痛苦" → "写作可控"**：字数预算 + 结构取舍，降低返工与自我怀疑

芋头的评价是：**受益匪浅，思路完全打开。**

---

## 07｜节目复盘：为什么我认为「AI 急诊时间」是有价值的？

因为它不是"教你几个 prompt"，而是做三件更实用的事：
1. **帮你定位问题根因**（写作痛点往往不在文笔）
2. **现场搭一套可复用流程**（你下次能自己跑）
3. **用可交付产物收尾**（文档库 + 成稿 + QA 结果）

但现实也很清楚：  
目前观众不多 + workshop 操作时间较长 → 互动数据不一定好看。  
要继续做下去，就必须把"参与入口"变得更明确。
